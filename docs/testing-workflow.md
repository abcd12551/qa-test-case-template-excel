# Testing Workflow

Use this basic workflow for manual QA execution:

1. Review requirements and acceptance criteria.
2. Prepare test cases in the template.
3. Add test data and environment details.
4. Execute test cases step by step.
5. Update Actual Result and Status.
6. Log defects for failed cases.
7. Re-test fixed defects and close execution.

## Status Guide

- Pass: Expected and actual results match.
- Fail: Expected and actual results do not match.
- Blocked: Test cannot proceed due to dependency.
- Not Run: Test is planned but not executed.
